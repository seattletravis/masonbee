# Security Policy

## Reporting a Vulnerability

If you discover a security issue in this project, please avoid posting it publicly.  
Instead, report it privately so it can be reviewed and addressed promptly.

You can contact the maintainer at:  
**<travis@lamberte.com>**

Please include:

- A clear description of the issue
- Steps to reproduce (if applicable)
- Any relevant logs, screenshots, or proof-of-concept details

You will receive a confirmation that the report was received, and we will work to resolve the issue as quickly as possible.

## Supported Versions

Security updates apply to the latest version of the project.  
Older versions may not receive patches.

## Security Features

This project uses GitHub’s built‑in security tooling, including:

- **CodeQL analysis** for static code scanning
- **Secret scanning** and **push protection**
- **Dependency vulnerability alerts**
- **Dependabot security updates**

These tools help identify and remediate potential issues before they reach production.

## Responsible Disclosure

We ask that all reporters follow responsible disclosure practices.  
Please give maintainers reasonable time to investigate and fix the issue before any public disclosure.
